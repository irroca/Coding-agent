# ADR 0008 — MCP client and server

Date: 2026-05-21
Status: Accepted (B4, B5)

## Context

Anthropic's **Model Context Protocol** (MCP) standardizes how LLM hosts and
tool/data servers talk over JSON-RPC. Claude Code, Cursor, Windsurf and
others all consume MCP servers. We want bidirectional interop:

1. **As a client**, Coding Agent should reuse existing MCP servers
   (filesystem, github, postgres, etc.) without having to reimplement each
   integration.
2. **As a server**, Coding Agent's nine built-in tools should be callable
   from any MCP-aware host so a Claude Code user can borrow `grep` /
   `read` / `edit` without leaving their editor.

## Decision

- Depend on the official `mcp` Python SDK (added as an optional `mcp` extra,
  pulled in by the `dev` extra so tests work).

### Client (`coding_agent/mcp/client.py`)

- `McpClientHost` owns the lifetime of stdio sessions to one or more
  configured external MCP servers (`Config.mcp.servers`).
- On startup it calls `initialize()` + `list_tools()` per server, synthesizes
  a `_RemoteTool` subclass per discovered tool, and registers it in the
  shared `_TOOL_REGISTRY` under the canonical name `mcp__<server>__<tool>`.
- `_RemoteTool` is declared `register=False` so subclass creation does not
  auto-register; the host registers deliberately after discovery and
  unregisters on shutdown.
- Pydantic `Params` is built from the remote tool's `inputSchema` with
  `extra="allow"`; we delegate field-level validation to the remote server.
- Default permission: `action="bash"` (treat unknown tools as bash-risk).
  Users who trust a particular server add a rule like
  `{tool: "mcp__github__*", decision: allow}`.

### Server (`coding_agent/mcp/server.py`)

- Console script: `coding-agent-mcp`. Also runnable as
  `python -m coding_agent.mcp.server`.
- Uses the low-level `mcp.server.Server` API with **explicit JSON schemas**
  generated from `Tool.schema()` — not FastMCP's type-inference path.
- Exposes every built-in tool *except* `task` (no LLM available inside an
  MCP server context).
- The permission engine and audit log still run on every call.
- `--list` flag dumps the JSON schema map to stdout for inspection without
  starting the stdio transport (used by the unit test).

## Consequences

- New tool ecosystem unlocked: any MCP server "just works" once configured.
- Coding Agent's tools become available inside Claude Code's MCP picker.
- Adds one optional dependency (`mcp>=0.9`).

## Alternatives considered

- **Roll our own JSON-RPC protocol** — rejected; MCP is becoming the
  cross-vendor standard and being part of that ecosystem is worth more
  than tailoring our own wire format.
- **FastMCP type-inference server** — rejected because our `Params` are
  already pydantic models with proper JSON Schema; round-tripping through
  FastMCP's signature inspector would lose annotations.
